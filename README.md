# 🧪 Tests Using Selenium

## 📌 Overview
This repository contains a collection of automated browser tests created using Selenium WebDriver with JavaScript.  
Each test is intentionally short and focused, designed to help me practice UI automation and understand how Selenium interacts with real web pages.

## 🎯 Purpose
- Practice browser automation using Selenium  
- Learn how to locate and interact with web elements  
- Understand waits, navigation, and user actions  
- Build a foundation for UI test automation  
- Create a portfolio piece demonstrating automation skills  

## 🛠️ Technologies Used
- Selenium WebDriver  
- JavaScript (Node.js)  
- ChromeDriver / WebDriver Manager  
- Mocha or Node test runner (depending on your setup)

## 🧩 Included Test Scenarios

### ✔ Basic Navigation Test
- Opens a webpage  
- Verifies the page title  
- Confirms navigation works correctly  

### ✔ Element Interaction Test
- Locates elements using CSS selectors or XPath  
- Performs actions such as click, type, submit  

### ✔ Form Submission Test
- Fills out input fields  
- Submits a form  
- Validates the result or confirmation message  

### ✔ Waits & Timing Test
- Uses implicit or explicit waits  
- Ensures elements are loaded before interacting  

## ▶️ How to Run Tests

### 1. Install dependencies


### 2. Run all tests

npm test

### 3. Run a specific test file

node tests/example.test.js


Or if using Mocha:
npx mocha tests/example.test.js


## 📘 Notes
- Tests are intentionally simple and focused on learning Selenium basics  
- Ideal for showcasing UI automation fundamentals in a portfolio  
- Can be extended with Page Object Model (POM) or more complex flows
